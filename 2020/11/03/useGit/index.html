Git 基本操作

Git 的工作就是创建和保存你项目的快照及与之后的快照进行对比。

本章将对有关创建与提交你的项目快照的命令作介绍。

Git 常用的是以下 6 个命令：git clone、git push、git add 、git commit、git checkout、git pull

说明

  • workspace：工作区
  • staging area：暂存区/缓存区
  • local repository：本地仓库
  • remote repository：远程仓库

一个简单的操作步骤：

git init
git add .
git commit

  • git init - 初始化仓库。
  • git add . - 添加文件到暂存区。
  • git commit -将暂存区内容添加到仓库中。

创建仓库命令

git init : 初始化仓库
git clone :拷贝一份远程仓库，也就是下载一个项目

提交与修改

Git 的工作就是创建和保存你的项目的快照及与之后的快照进行对比。
以下列出了有关创建与提交你的项目的快照的命令：

git add : 添加文件到仓库
git status : 查看仓库当前的装填，显示有变更的文件
git diff : 比较文件的不同，即暂存区和工作区的差异
git commit : 提交暂存区到本地仓库
git reset : 回退版本
giit rm : 删除工作区文件
git mv : 移动或重命名工作区文件

提交日志

git log : 查看历史提交记录
git blame：以列表形式查看指定文件的历史修改记录

远程操作

git remote ：远程仓库操作
git fetch :从远程获取代码库
git pull : 下载远程代码并合并
git push : 上传远程代码并合并

Git版本控制Demo

安装Git以后，在F盘根目录下右键–>Git bush here
依次执行以下命令：

mkdir gitCode
cd gitCode
git init
touch test.txt

第一步：使用命令添加到暂存区里面去。如下：

git add readme.txt

没有任何提示，说明已经添加成功了。

第二步：用命令 git commit告诉Git，把文件提交到仓库。

git commit

第三步：打开test.txt 文本文件，随意添加内容

第四步：将修改过的文件提交到仓库

git add .
git commit -m "v1.0"

第五部：查看版本记录

git log

第六步：选择回退的版本 这里选择版本号为：3e039b7 的

git reset --hard 3e039b7

此时打开test.txt文件应该是没有内容的

如果是回退到上一个版本，还可以用命令：

git reset --hard HEAD^
